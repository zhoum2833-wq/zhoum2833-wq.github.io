# AI 基础知识

你可能听过很多词：API Key、Token、上下文窗口、多模态、Agent……但它们到底是什么意思？这篇文章帮你用最简单的方式理清楚。

## API Key：你的 AI 密码

API Key 就是一串字母和数字组成的"身份令牌"。它有两个作用：

1. **身份认证**：告诉 AI 服务商"是我在用"
2. **费用结算**：用了多少 token 就扣多少钱

API Key 就像银行卡密码——不能公开，不能上传到 GitHub。如果不小心泄露了，别人刷你的额度，账单还是你付。

::: danger 保护你的 API Key
- 永远不要把 API Key 写在代码里
- 用环境变量或配置文件存储，并在 .gitignore 中忽略它们
- 如果怀疑泄露，立刻去控制台重置
:::

## Token：AI 的"最小读取单位"

Token 不是"一个汉字"也不是"一个单词"。它是 AI 处理文本的**最小粒度**。大致可以这样估算：

| 文本 | 大约 Token 数 |
|------|---------------|
| 1 个中文字符 | ~2 token |
| 1 个英文单词 | ~1.3 token |
| 1 行代码（如 `int x = 0;`） | ~5-8 token |
| 一本 300 页的书 | ~250,000 token |

为什么中文更"贵"？因为中文字符的信息密度高，一个字符往往等于英文好几个词的意思，但 tokenizer 按照编码方式切割，中文每个字符占更多 token。

## 上下文窗口：AI 的短期记忆

上下文窗口决定了 AI 能"记住"多少对话内容。每当你发送一条消息，AI 实际上是把**整个对话历史**重新读一遍。

| 模型 | 上下文窗口 | 类比 |
|------|------------|------|
| GPT-3.5（初代） | 4K token | 一篇短文 |
| GPT-4（早期） | 8K-128K token | 一篇中篇小说 |
| DeepSeek V4 Pro | 1,000K token | 三本《三体》 |

::: tip 上下文窗口的真正价值
大上下文窗口不仅让你能和 AI 长聊，更重要的是**可以把整个项目的代码都放进去**。AI 能同时看到 main.c、所有 bsp 模块、Makefile，然后基于整体理解给出建议，而不是盲人摸象。
:::

## 多模态（Multimodal）

早期 AI 只能处理纯文本。多模态 AI 可以同时处理多种信息形式：

- **文本** → 读写文字
- **图像** → 看懂图片（比如电路原理图、示波器截图、实物照片）
- **音频** → 听懂声音、识别语音
- **视频** → 未来的趋势

你使用的 DeepSeek V4 Pro 本身是一个纯文本模型，但通过 **Skills 机制**（参见下一章）可以获得图像理解等额外能力。这就像一个近视的人戴上眼镜——模型本身"看不到"图片，但 Skill 帮它"看清"了。

## Skills：AI 的"外挂插件"

Skill 是给 AI 装上的"额外技能包"。每个 Skill 赋予 AI 一个原本不具备的能力：

| Skill 名称 | 功能 |
|-----------|------|
| deepseek-vision | 让纯文本模型看懂图片 |
| docx | 生成和编辑 Word 文档 |
| pdf | 读取和处理 PDF 文件 |
| xlsx | 处理和生成电子表格 |
| pptx | 创建演示文稿 |

## Agent：能独立完成任务的 AI

普通的聊天 AI 是一问一答。Agent（智能体）不同——你给它一个目标，它能**独立规划并执行多步操作**。比如：

- 你说"帮我在项目里添加一个 OLED 驱动"→ Agent 会：搜索现有 I2C 代码 → 设计接口 → 生成代码 → 修改 Makefile → 测试编译
- 你什么都不用管，等结果就行

不同 Agent 有不同的专长。Explore Agent 擅长搜索代码，general-purpose Agent 什么都能做。

::: warning
Agent 虽然强大，但它仍然可能出错。Agent 生成的代码你依然需要审查——它只是在"执行"，判断力最终还是在你手里。
:::

## 小结

AI 不是魔法——它本质是一个"超级猜词器"，根据上下文预测下一个最可能的 token。token 不是字，上下文窗口有上限，训练和推理是两码事——搞清这三个概念，AI 的"幻觉"就不再神秘了。
