# Skills 与多模态

DeepSeek V4 Pro 是一个纯文本模型——它"生来"只能处理文字。但通过 Skills 机制，它可以获得看图、写文档、处理表格等额外能力。这就像给智能手机装 App，每个 App 赋予手机一种新功能。

## 核心场景：看示波器波形

假设你用示波器测了 PWM 信号，拍了一张照片。你想问 AI："这个波形正常吗？占空比大概是多少？"

- **没有 deepseek-vision skill**：AI 会回答"我无法查看图片"
- **有 deepseek-vision skill**：AI 会调用视觉模型分析图片，然后告诉你波形的周期、占空比、是否有毛刺

这就是 Skill 的价值——让纯文本模型"长出了眼睛"。

## 怎么使用 Skill

Skill 不需要你手动"打开"或"关闭"。当你的对话上下文触发某个 Skill 的能力范围时，系统会自动调用。比如你贴了一个图片路径，deepseek-vision 就会被触发。

::: tip
Skill 的执行是**透明的**——你不需要知道底层调用的是哪个视觉模型，只需要知道"AI 现在能看图片了"。
:::

## 常用 Skill 速览

| Skill | 解决的问题 | 典型使用场景 |
|-------|-----------|-------------|
| deepseek-vision | 纯文本模型无法看图 | 分析电路照片、示波器截图、PCB 布局 |
| docx | AI 不能直接生成 Word 文档 | 撰写比赛报告、生成技术文档 |
| pdf | 从 PDF 中提取信息 | 读取芯片数据手册、参考设计 |
| xlsx | 处理和生成表格 | 整理传感器测试数据、绘制图表 |
| pptx | 创建演示文稿 | 准备比赛答辩 PPT |

## 实际案例

**场景 1：调试电路板**

你拍了一张自己焊的 PCB 照片，问："帮我看看这个焊点有没有问题？"

deepseek-vision skill 会分析图片，指出哪里可能虚焊、锡量不足，或者引脚间是否有连锡风险。

**场景 2：阅读数据手册**

TI 的 MSPM0G3507 数据手册是 PDF 格式，几百页。你可以直接让 AI 从 PDF 中提取特定信息：

> "从这份数据手册中，帮我找 TIMG 定时器的时钟源选择和分频配置。"

pdf skill 会读取 PDF 内容，AI 再从中提取并总结相关信息。

**场景 3：生成比赛文档**

比赛结束后需要提交报告。你可以告诉 AI：

> "根据我们的项目代码，生成一份 Word 文档，包含系统架构图描述、各模块功能说明、关键算法解释。"

docx skill 会帮你排版成规范的 Word 文档。

## 类比：近视眼镜

## 小结

Skills 是 DeepSeek V4 Pro 的"感官系统"——纯文本模型本身看不到图、听不到声，通过 deepseek-vision 等 Skill 补上这些能力。知道哪些关键词触发哪个 Skill，AI 就能在合适的场景自动"戴上眼镜"或"拿起笔"。

> **纯文本模型 + deepseek-vision = 近视的人戴上眼镜**

模型本身没有"视觉皮层"，但通过 Skill 接入视觉模型后，它能够"看见"并描述图片内容。对你来说，使用体验就是——AI 突然能看图了。

::: warning
Skills 虽然强大，但每个 Skill 都是独立的能力模块。别指望 AI 看完你的 PCB 照片就能帮你画原理图——它只能"看懂"，不能直接转换为电路设计动作，除非你再调用其他工具。
:::
