# 数据是如何流动的

在单片机系统里，数据从传感器出发，经过一系列处理，最终驱动电机或点亮 LED。理解数据流动的路径，是理解整个嵌入式系统的关键。

## 核心观点：CPU 是交通枢纽

在绝大多数情况下，**CPU 是所有数据的转运中心**。传感器数据要进 CPU 处理，电机命令要从 CPU 发出。唯一的例外是 DMA（直接存储器访问），它可以在 CPU 不参与的情况下搬运数据——后面章节会详细讲。

## 三条典型数据路径

### 路径一：传感器 → 电机

```
[红外传感器] ──模拟电压──▶ [ADC] ──数字值──▶ [RAM 变量]
                                                    │
                                                    ▼
                                              [CPU 计算]
                                              (偏差值 = 目标值 - 当前值)
                                                    │
                                                    ▼
                                            [PWM 寄存器]
                                                    │
                                                    ▼
                                              [电机驱动]
                                                    │
                                                    ▼
                                              [电机转动]
```

这是最常见的控制路径。红外传感器输出一个模拟电压（比如 0~3.3V），ADC（模数转换器）把它变成数字（比如 0~4095）。这个数字存到 RAM 的一个变量里。CPU 读取这个变量，算出偏差，再算出该给电机多大的力。最后 CPU 把计算结果写入 PWM 寄存器，PWM 寄存器控制电机驱动芯片，电机转动。

### 路径二：按键 → LED

```
[按键按下] ──电平变化──▶ [GPIO 引脚] ──触发──▶ [EXTI 中断]
                                                      │
                                                      ▼
                                                [CPU 执行 ISR]
                                              (中断服务函数)
                                                      │
                                                      ▼
                                                [GPIO 输出寄存器]
                                                      │
                                                      ▼
                                                  [LED 亮]
```

按键按下，引脚电平从高变低（或从低变高）。这个变化触发 EXTI（外部中断），CPU 暂停当前工作，执行中断服务函数（ISR）。ISR 里往 GPIO 输出寄存器写一个值，LED 就亮了。整个过程在微秒级别完成，你感觉就是"按下去灯就亮"。

### 路径三：陀螺仪 → 上位机遥测

```
[陀螺仪] ──SPI/I2C──▶ [UART] ──串行数据──▶ [DMA]
                                              │
                                              ▼
                                         [RAM 数组]
                                              │
                                              ▼
                                        [CPU 姿态解算]
                                      (融合加速度+角速度)
                                              │
                                              ▼
                                         [UART 发送]
                                              │
                                              ▼
                                    [Vofa+/上位机显示]
```

姿态传感器通过 SPI 或 I2C 接口不断产生数据。这些数据可以被 DMA 直接搬运到 RAM 的一个数组里（CPU 不参与！）。等一帧数据凑齐了，CPU 执行姿态融合算法（比如互补滤波或卡尔曼滤波），算出小车的角度、角速度。最后通过 UART 把结果发给上位机软件（比如 Vofa+），上位机上画出实时波形。

## 寄存器详解

寄存器是单片机中最重要的概念之一。它是一个**特殊的内存位置**，但它不在 RAM 里，而在外设模块内部。

把寄存器想象成**控制面板上的开关和仪表**：

- **写寄存器 = 拨动开关**（下命令）：比如 `TIM3->CCR2 = 500;` 意思是"定时器 3 的通道 2 输出比较值设为 500"，这条命令直接改变了 PWM 的占空比。
- **读寄存器 = 看仪表**（收数据）：比如 `uint8_t status = USART1->SR;` 意思是"看看串口 1 的状态寄存器"，通过读这个值你能知道数据有没有发完、有没有收到新数据。

```c
// 写寄存器：控制硬件
GPIOA->BSRR = GPIO_PIN_5;   // PA5 输出高电平 → 外接的 LED 亮了

// 读寄存器：从硬件获取信息
if (USART1->SR & USART_SR_RXNE) {   // 检查串口收到数据没有
    uint8_t data = USART1->DR;      // 有数据，读出来
}
```

::: tip
**寄存器 vs 普通变量**：普通变量存在 RAM 里，只存储数值。寄存器在硬件模块里，读写它会触发硬件动作——写 GPIO 寄存器会改变引脚电平，读 UART 寄存器会取出接收到的数据。寄存器就是软硬件之间的**桥梁**。
:::

## 数据流动的主要通道

| 通道 | 全称 | 作用 | 类比 |
|------|------|------|------|
| GPIO | 通用输入输出 | 读按键、控制 LED | 电灯开关 |
| ADC | 模数转换器 | 把模拟电压变成数字 | 温度计读数 |
| UART | 通用异步收发器 | 串口通信（与上位机） | 对讲机 |
| SPI | 串行外设接口 | 高速通信（传感器） | 专用快递通道 |
| I2C | 集成电路间总线 | 低速多设备通信 | 小区广播 |
| PWM | 脉宽调制 | 控制电机转速 | 水龙头调节水流 |
| DMA | 直接存储器访问 | 自动搬运数据 | 传送带 |

## 哪些数据流过 CPU？哪些不流？

| 场景 | 数据路径 | CPU 参与？ |
|------|----------|-----------|
| 按键检测（轮询） | GPIO → CPU → 逻辑判断 | 是 |
| 按键检测（中断） | GPIO → EXTI → CPU(ISR) | 是 |
| ADC 单次采样 | ADC → CPU → RAM | 是 |
| ADC 连续采样 + DMA | ADC → DMA → RAM | **否** |
| UART 发送单字节 | CPU → UART 数据寄存器 | 是 |
| UART 大批量接收 + DMA | UART → DMA → RAM | **否** |
| PWM 输出 | CPU 写寄存器一次，硬件自动持续输出 | **半参与** |

::: warning
**DMA 不是万能的**。DMA 只能做"从 A 搬到 B"这种简单重复的工作。任何需要计算、判断、决策的事情，仍然必须 CPU 亲自处理。DMA 负责搬运，CPU 负责思考。
:::

## 小结

数据在单片机里的流动就像城市交通：传感器是起点，电机/LED 是终点，CPU 是交通指挥中心。GPIO、ADC、UART、SPI 是数据进出的大门，DMA 是自动驾驶的货车——帮你搬数据，但不帮你做决策。理清这些路径，系统的血液循环就一目了然了。
