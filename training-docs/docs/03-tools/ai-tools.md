# AI 辅助嵌入式开发

**核心原则：人负责架构、规范、逻辑；AI 负责落地编码、模板实现、批量重复工作**

一、Claude Code

核心用法：写CLAUDE.md 
CLAUDE.md 是给 AI 看的全局项目手册，让 AI 永久适配当前工程架构，不用每次重复解释。内容包含：
- 项目分层架构（裸机三层 / RTOS四层、层级调用规则）
- 完整硬件信息（芯片型号、引脚分配、I2C/SPI模式、外设参数）
- 统一编码规范（文件结构、函数命名、注释格式、禁止跨层调用）
- 系统初始化顺序、编译烧录流程
- 历史踩坑记录、硬件禁忌、易错时序问题

二、Cursor

核心用法：IDE 实时跟随，局部编码、即时纠错
Cursor 深度嵌入代码编辑器，适合日常高频开发，不用全局重构，只做局部精准落地。
主要用途：
- 业务逻辑代码补全、状态机编写、算法实现
- 实时修复编译报错、语法错误、逻辑漏洞
- 单行/单文件优化、注释补齐、代码精简
核心优势：能实时识别当前代码上下文、项目风格，手写代码的同时智能补全，大幅提升日常编码速度，适合写 APP 业务、任务逻辑。

三、OpenAI Codex

核心用法：批量标准化生成、自动化工具开发
Codex 擅长云端批量处理，不适合精细架构重构，适合做“标准化重复工作”。
主要用途：
- 批量生成 BSP 驱动模板、工程框架、初始化代码
- 批量补全注释、统一代码格式、生成测试用例
- 编写自动化脚本：编译、烧录、日志解析、数据处理工具
核心优势：一次性批量产出标准化代码，解放重复造轮子的工作，让项目快速搭好基础框架。
