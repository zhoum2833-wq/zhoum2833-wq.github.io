# 开发工具全景

你面前有一堆名字看起来像缩写的东西——Keil、HAL、CMSIS、VOFA……到底每个是干什么的？它们怎么串成一条完整的开发流水线？这篇文章帮你把工具全景画出来，让你知道每样东西在开发流程中处于哪个位置。

以我们的 **4WD_Car** 四轮小车项目为例，开发工具链分四块：

## VS Code + AI：写代码的地方

VS Code 是微软的免费代码编辑器，用它来写 C 代码、浏览工程文件。更重要的是，它让你能跟 AI（比如 Claude Code）结对编程——你描述需求，AI 帮你写代码、查 bug、解释逻辑。

为什么不直接用 Keil5 写代码？Keil5 的编辑器功能有限，没有 AI 插件，代码补全也弱。VS Code 才是真正的"写字台"。

## Keil5：编译和烧录

Keil5（MDK-ARM）是 KEIL 公司（现属 ARM）的 IDE。它的任务就两个：**编译**（把你的 C 代码变成单片机认识的机器码）和**烧录**（把机器码写进 Flash）。当然，调试也靠它——断点、单步、看变量值。

在 4WD_Car 项目中，Keil5 工程文件位于 `Project/4WD_Car.uvprojx`。VS Code 里写完代码 → 切到 Keil5 按 F7 编译 → 按 F8 烧录。

## VOFA+：无线遥测监视

单片机没有屏幕，你怎么知道它内部在算什么？VOFA+ 是一个串口调试/遥测工具。4WD_Car 通过蓝牙串口（USART6）把数据无线发给电脑上的 VOFA+，你可以看到实时的**波形图**——电机速度曲线、编码器计数、IMU 角度、PID 输出。

没有这个，你就是在"盲调"。有了它，你就是在"开着仪表盘开车"。

## Python 脚本：离线辅助工具

有些活不适合在单片机上做——比如训练一个数字识别模型、批量标注图片、把 PyTorch 模型转成 ONNX。这些都在电脑上用 Python 搞定。

4WD_Car 的 `tools/` 目录里就是这些脚本：采集数据集、训练 CNN、导出模型给 Maxicam 视觉模块用。

## 实际工作流

```
VS Code + Claude Code（写代码 + AI 辅助）
        ↓
Keil5 打开 Project/4WD_Car.uvprojx（编译 F7 + 烧录 F8）
        ↓
VOFA+ 连接蓝牙串口（观察遥测波形、调试）
        ↓
Python 脚本 tools/*.py（离线数据处理、模型训练）
```

四块工具各司其职。本文后面几节会详细介绍 Keil5 的用法、工程结构的设计、AI 编程的实践、以及 VOFA 和 Python 工具链。
