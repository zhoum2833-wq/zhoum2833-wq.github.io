# 开发工具全景

你面前有几十个软件，每个软件的名字看起来都像缩写——CubeMX、Keil、HAL、CMSIS……到底每个是干什么的？它们怎么串成一条完整的开发流水线？这篇文章帮你把工具全景画出来，让你知道每样东西在开发流程中处于哪个位置，不再被缩写淹没。

电赛开发的工具链分三块：

## CubeMX：生成初始化代码

用图形界面配置时钟、引脚和外设参数，一键生成 Keil 工程和初始化代码。

## Keil5：编译和烧录

Keil5 是 KEIL 公司（现属 ARM）的 IDE。免费社区版支持 32KB 以内的代码，竞赛够用。

直接打开 CubeMX 生成的 `.uvprojx` 工程文件，点编译、点下载就完了。不需要自己搭 GCC 工具链——电赛抢时间，能跑就行。

## VS Code：写代码 + AI 辅助

VS Code 是微软的免费代码编辑器。用它来编辑 C 代码、浏览工程文件、和 AI（Claude Code）结对编程。

为什么不直接用 Keil5 写代码？Keil5 的编辑器功能有限，没有 AI 插件，界面也不如 VS Code 方便。

## 实际工作流

```
CubeMX 配引脚/时钟 → 生成 Keil 工程
                           ↓
VS Code 打开工程文件夹，写 C 代码，AI 辅助
                           ↓
Keil5 打开 .uvprojx，编译、下载、调试
```

三个工具各司其职，不折腾开源工具链。

## 小结

新手入门嵌入式开发，工具选择是最容易踩的坑。本节的核心就一句话：**CubeMX 负责"生成"、VS Code 负责"写"、Keil5 负责"编译下载"** —— 就像装修队（CubeMX）给你搭好毛坯房的水电线路，你（VS Code）在里面设计和装修，最后施工队（Keil5）把房子盖起来。记住这个三角分工，不要让任何一个工具越俎代庖。下一节我们会拆开 CubeMX 的工程结构，告诉你哪些文件是你的地盘、哪些是禁区。
