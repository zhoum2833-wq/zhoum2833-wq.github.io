# 开发工具全景

## 什么是 IDE？

你要给单片机写程序，需要把一行行 C 语言代码变成单片机能执行的机器指令（一系列 0 和 1）。完成这个过程的工具套装叫 **IDE**（Integrated Development Environment，集成开发环境）。

IDE 是一个"瑞士军刀"，里面包含四样东西：

1. **编辑器（Editor）**：写代码的地方。有语法高亮（关键字变色）、自动补全、错误提示等功能。
2. **编译器（Compiler）**：把你的 C 代码翻译成单片机认识的机器指令。这个过程叫"编译"。
3. **调试器（Debugger）**：在程序运行时暂停、单步执行、查看变量值，帮你找 bug。
4. **下载器（Programmer/Flash Loader）**：把编译好的程序烧录（写入）到单片机的 Flash 存储器里。

简单说：**在编辑器里写代码 → 编译器翻译 → 下载器烧录 → 调试器帮你改错**。

## 两大阵营：Keil5 vs VS Code + GCC

嵌入式开发工具已经演化了三十多年，目前主要有两个流派：

### Keil MDK-ARM（简称 Keil5）

Keil 是嵌入式开发界的"老大哥"，在 STM32 生态中统治了十几年。正点原子、野火、普中等几乎所有中文教程都以 Keil5 作为默认 IDE。

**优点**：
- **高度集成**：安装即用，编译器、调试器、下载器全部内置，不需要额外配置。
- **官方支持**：ST 官方例程很多以 Keil 格式发布，CubeMX 原生生成 Keil 工程。
- **调试功能强**：配合 J-Link/ST-Link，调试体验流畅。
- **资料丰富**：遇到问题几乎都能搜到 Keil 版本的解答。

**缺点**：
- **收费**：商业授权几千元。社区免费版限制编译出的代码不超过 **32KB**。
- **界面老旧**：Keil5 的 UI 停留在 Windows XP 时代，不支持现代编辑器的很多功能。
- **封闭生态**：扩展性差，没有 AI 辅助插件，不支持 Git 可视化。
- **Windows 独占**：Mac 和 Linux 用户无法使用。

### VS Code + GCC 工具链

这是新一代嵌入式开发的主流方式。VS Code 是微软开发的免费代码编辑器，GCC（GNU Compiler Collection）是开源免费的编译器。

**优点**：
- **完全免费**：没有代码量限制。
- **现代化编辑器**：界面美观，扩展生态极其丰富（主题、快捷键、Git 集成、远程开发等）。
- **AI 友好**：GitHub Copilot、各种 AI 编程助手都有 VS Code 插件。
- **跨平台**：Windows / Mac / Linux 都能用。
- **能力可迁移**：VS Code 不只是嵌入式能用，Web 开发、Python、Java 都能用。学会一个工具，受用终身。

**缺点**：
- **需要自己搭建**：装完 VS Code 不等于能开发 STM32，还需要安装 GCC、配置 Makefile、设置调试器……对新手来说有一定门槛。
- **资料相对少**：大多数中文教程仍然是 Keil 版本。遇到 Makefile 或链接脚本的问题需要自己排查。
- **调试不如 Keil 直观**：虽然能调，但配置和操作都比 Keil 麻烦一些。

## 两种方案的详细对比

| 对比维度 | Keil MDK-ARM (Keil5) | VS Code + GCC |
|---------|---------------------|---------------|
| 费用 | 商业版 ~4000元；免费版限32KB | 完全免费 |
| 代码量限制 | 免费版 32KB | 无限制 |
| 界面 | 老旧，Windows XP 风格 | 现代，美观 |
| AI 集成 | 基本没有 | 完善（Copilot 等） |
| 跨平台 | 仅 Windows | Windows/Mac/Linux |
| 上手难度 | 安装即用，低 | 需要配置环境 |
| 调试体验 | 优秀，开箱即用 | 良好，需配置 |
| 社区资料 | 非常丰富（中文） | 英文为主，中文在增长 |
| 可持续性 | 走下坡路 | 未来趋势 |
| 技能可迁移性 | 只能嵌入式 | 全栈开发通用 |

## 我们的选择

**本项目推荐使用 VS Code + GCC。**

理由很简单：
1. **完全免费**，没有 32KB 限制。
2. 提供的 **Makefile 模板**已经帮你配好了编译和下载流程，降低了入门门槛。
3. VS Code 的编辑体验远超 Keil5，配合 Copilot 写代码效率极高。
4. 学会 VS Code 是长期投资，这个技能在所有编程领域都能用。

::: tip 你需要知道 Keil5 的存在
比赛中可能出现以下情况：
- 队友只会用 Keil5。
- 赛方提供的例程是 Keil 格式。
- 某些老旧教程的配置步骤只有 Keil 版本。

所以你需要**认识 Keil5 的界面**，知道怎么打开一个 Keil 工程、怎么编译、怎么下载。但日常开发用 VS Code 就够了。
:::

## 工具链全景图

一个完整的嵌入式开发流程涉及以下工具：

```
              CubeMX (配置时钟和引脚)
                  ↓ 生成代码
VS Code (写 C 代码) → arm-none-eabi-gcc (编译器) → .elf 文件
                                                     ↓
                                            arm-none-eabi-objcopy (格式转换)
                                                     ↓
                                                  .hex / .bin 文件
                                                     ↓
                                            OpenOCD / pyOCD (下载工具)
                                                     ↓
                                            ST-Link / DAP-Link (调试器硬件)
                                                     ↓
                                            单片机 Flash 存储器
```

::: warning 不要被这么多工具吓到
虽然流程看起来长，但 Makefile 已经帮你自动化了编译和下载。你每天的实际操作只有两步：
1. 在 VS Code 里写代码。
2. 按 `Ctrl+Shift+B` 编译，或直接在终端输入 `make flash` 一键编译+下载。
:::

## 小结

Keil5 是嵌入式开发的传统工具，资料丰富但收费且老旧。VS Code + GCC 是现代替代方案，免费、强大、未来可期。我们选择 VS Code + GCC，Makefile 模板已经帮你做了大部分配置工作，你只需要专心写代码。
