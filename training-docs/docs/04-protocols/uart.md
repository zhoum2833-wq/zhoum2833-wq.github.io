# UART —— 最基础的串口通信

## 什么是 UART？

**UART**（通用异步收发器）是嵌入式里最简单，最常用的通信方式。两个设备之间只需要**两根线**（外加一根公共地线）就能互相发送数据。

- **TX**（Transmit，发送）：数据从本设备送出。
- **RX**（Receive，接收）：数据从本设备接收。
- **GND**（Ground，地线）：提供一个共同的电压参考点。

连接规则非常简单：**A 的 TX 接 B 的 RX，A 的 RX 接 B 的 TX。交叉连接。**

```
设备 A                    设备 B
  TX ──────────────────→ RX
  RX ←────────────────── TX
 GND ─────────────────── GND
```

## "异步"是什么意思？

通信分为两大类：**同步**和**异步**。

- **同步通信**：除了数据线，还有一根**时钟线**。发送方在时钟线上一拍一拍地打节拍，接收方跟着节拍读取数据。就像跳舞时有人喊"一、二、三、四"。
- **异步通信**：**没有时钟线**。双方提前约定好一个共同的"节奏"（波特率），然后各自用自己的时钟来发送和接收数据。

UART 是异步的——没有时钟线，双方靠**事先约定好的波特率**来同步。这意味着双方必须严格用相同的波特率，否则数据就会变成乱码。

## 一个字节是怎么被发送出去的？

假设你要发送一个字节的数据 `0x41`（二进制是 `01000001`，也就是 ASCII 字符 `'A'`），配上最常见的配置 **8-N-1**（8 个数据位、无校验位、1 个停止位），TXD 线上的电平变化如下：

```
空闲状态（高电平）
  ↓
  |‾‾‾‾‾‾‾|__|‾‾|__|__|__|__|__|‾‾|__|‾‾‾‾‾‾‾‾‾|
  |  Start | D0 D1 D2 D3 D4 D5 D6 D7 | Stop  | 空闲
  |   0    | 1  0  0  0  0  0  1  0  |   1   |
  |        |        0x41 (LSB first)  |       |
```

逐段解释：

1. **空闲状态**：线路上保持高电平（逻辑 1）。双方知道此时没有数据在传输。
2. **起始位（Start Bit）**：发送方把线路拉低一个比特周期。接收方检测到这个下降沿，"哦，数据来了！"，开始准备接收。
3. **数据位（Data Bits）**：紧接着起始位之后是 5 到 9 个数据位（最常见的是 8 位）。**最低位先发送**（这叫 LSB First，Least Significant Bit first）。上图中 `D0` 是最低位。
4. **校验位（Parity Bit，可选）**：如果开启了校验，数据位之后会多一个校验位，用来检测传输中是否出现了错误。8-N-1 的 "N" 就是 No parity，不校验。
5. **停止位（Stop Bit(s)）**：数据发送完毕，发送方把线路拉回高电平并保持 1 到 2 个比特周期。这是一个字节传输结束的标志。

## 常见配置：8-N-1

99% 的场景下，使用的都是 **8-N-1** 配置：

| 参数 | 值 | 含义 |
|------|-----|------|
| 数据位 | 8 | 一个字节就是 8 位，刚好对应一个 `char` |
| 校验位 | None (N) | 不做奇偶校验 |
| 停止位 | 1 | 1 个停止位 |

**UART 是嵌入式开发中最常用的通信方式**

## 接收数据的两种方式

UART 接收数据有三种常见方式：

1. **阻塞接收（Blocking）**：调用 `HAL_UART_Receive()`，CPU 一直等着，直到收到指定数量的数据才继续往下执行。适合简单应用，但等数据期间 CPU 什么别的也干不了。
2. **中断接收（Interrupt）**：调用 `HAL_UART_Receive_IT()`，开启中断。数据到达时硬件触发中断，CPU 来处理。**最常用的方式。**
3. **DMA 接收（Direct Memory Access）**：数据自动存到内存，完全不占用 CPU。适合高速、大批量的数据接收。
