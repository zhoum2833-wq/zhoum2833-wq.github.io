# I2C —— 两线制主从通信

## 什么是 I2C？

**I2C**（Inter-Integrated Circuit，集成电路间总线）是一种只用**两根线**就能连接**很多个设备**的通信协议。它由飞利浦（现在的 NXP）在 1982 年发明，最初是为了让电视机里不同芯片之间方便通信。

两根线分别是：
- **SCL**（Serial Clock，串行时钟线）：由主机（MCU）产生，为通信提供节拍。
- **SDA**（Serial Data，串行数据线）：承载实际的数据，双向传输。

::: tip 类比：课堂点名
想象一个课堂，老师（主机/Master）拿着花名册，想找某个学生（从机/Slave）回答问题。

老师喊："张三！" → 这就是在总线上广播地址。
只有张三站起来回答："到！" → 这就是从机响应。
其他学生虽然听到了老师的喊声，但因为叫的不是自己，所以保持安静。
老师问："把第 3 题的答案告诉我。" → 发送命令。
张三回答："答案是 42。" → 返回数据。

I2C 就是这样工作的——主机先喊"地址"，只有对应地址的设备才回应。
:::

## 硬件连接：两根线 + 两个电阻

I2C 总线的物理连接非常简单：

```
         VCC (3.3V)
          │
          ├──── 电阻 R1 (4.7kΩ) ──── SDA 线 ────┬─────┬─────┬─────
          │                                     │     │     │
          ├──── 电阻 R2 (4.7kΩ) ──── SCL 线 ────┬─────┬─────┬─────
          │                                     │     │     │
         GND                                  MCU  设备A  设备B
                                            (主机) (从机) (从机)
```

两个电阻 R1 和 R2 叫**上拉电阻**（Pull-up Resistor），通常取 **4.7kΩ**。

为什么需要上拉电阻？
I2C 的设备只能把线路**拉低**到 GND（输出 0），但不能主动把线路**拉高**到 VCC（输出 1）。线路变高靠的是上拉电阻把电压"拽"上去。这种设计叫**开漏输出**（Open-Drain），好处是多个设备可以共享同一条线而不会互相冲突——因为大家只拉低不拉高，不会出现"一个设备输出高、另一个输出低导致短路"的情况。

::: warning 忘记加上拉电阻
如果你自己画 PCB 或者用面包板搭 I2C 电路，**千万不要忘记加上拉电阻**。没有上拉电阻，SCL 和 SDA 的电平处于悬浮状态，通信会失败。有些开发板在板上已经集成了上拉电阻，大部分传感器模块也自带——但不是所有的都有。不确定的时候用万用表测一下。
:::

## 主从架构和设备地址

I2C 是**主从（Master-Slave）架构**：
- **主机（Master）**：只有一个，通常是你的 MCU。主机发起所有通信，产生时钟信号。
- **从机（Slave）**：可以有很多个，每个都有一个唯一的 **7 位地址**（或 10 位地址，但 7 位最常见）。

每个 I2C 从机（传感器、存储器、显示屏等）在出厂时就被分配了一个或多个地址。数据手册上会写清楚。

常见的 I2C 设备地址：

| 设备类型 | 常见型号 | 7 位地址 |
|---------|---------|----------|
| OLED 显示屏 | SSD1306 | 0x3C（或 0x3D） |
| 温湿度传感器 | SHT30 | 0x44 |
| 六轴陀螺仪 | MPU6050 | 0x68（或 0x69） |
| EEPROM 存储器 | AT24C02 | 0x50 |
| I/O 扩展芯片 | PCF8574 | 0x20 ~ 0x27 |

## 通信时序：一次完整的 I2C 对话

下面是一次完整的 I2C 通信过程，主机向从机写一个字节数据：

```
SDA: ‾‾\___/‾‾\_/‾‾\_____/‾‾\_____/‾‾\_/‾‾\_/‾‾\_/‾‾\___/‾‾\_/‾‾\____
SCL: ‾‾‾\__/‾‾\_/‾‾‾\___/‾‾‾\___/‾‾‾\_/‾‾‾\_/‾‾‾\_/‾‾‾\___/‾‾‾\_/‾‾‾
     |Start| A6 A5 A4 A3 A2 A1 A0 R/W|ACK| D7 D6 D5 D4 D3 D2 D1 D0|ACK|Stop
     |     |＜──  7位地址  ──＞| 0  |   |＜──  1字节数据  ──＞|   |
```

逐阶段解释：

1. **起始条件（Start Condition）**：SCL 为高电平时，SDA 从高变低。主机宣告"通信开始，大家注意！"
2. **地址 + 读写位**：主机发送 7 位从机地址，再跟 1 位读写方向（0 = 主机写数据给从机，1 = 主机从从机读数据）。
3. **应答位（ACK）**：第 9 个时钟脉冲时，主机释放 SDA，如果从机认出了自己的地址，就把 SDA 拉低表示"我在！"（ACK）。如果没有任何从机响应，SDA 保持高电平（NACK）。
4. **数据传输**：接下来是实际的数据，每个字节（8 位）后面都跟着一个应答位。
5. **停止条件（Stop Condition）**：SCL 为高电平时，SDA 从低变高。主机宣告"通话结束"。

## 为什么两根线能连很多设备？

这就是地址机制的妙用。总线上所有设备都"听"着总线上的地址信息。主机广播一个地址后：
- 地址匹配的那个从机：回应 ACK，开始收发数据。
- 地址不匹配的所有从机：继续沉默，忽略后续数据。

所以 I2C 用两根线连接 100+ 个设备从物理上是可行的（实际上受限于总线电容和地址空间）。

::: danger 地址冲突
如果你挂两个相同型号的传感器在同一个 I2C 总线上，而且它们有相同的地址，就会出现冲突——两个设备同时回应，数据乱套。解决办法：
1. 看数据手册是否能通过硬件引脚改变地址（很多芯片有 ADDR 引脚）。
2. 用 I2C 多路复用器（如 TCA9548A）。
3. 用 MCU 的另一个 I2C 外设接另外的传感器。
:::

## I2C 的几种速度

| 模式 | 速度 | 说明 |
|------|------|------|
| Standard（标准） | 100 kbps | 最慢，兼容性最好 |
| Fast（快速） | 400 kbps | **最常用**，几乎所有设备都支持 |
| Fast Plus（快速+） | 1 Mbps | 需要设备和 MCU 都支持 |
| High Speed（高速） | 3.4 Mbps | 很少用，需要特殊硬件 |

## 代码示例：I2C 扫描总线上的设备

下面这个代码非常实用——它可以扫描 I2C 总线上挂载了哪些设备，并打印它们的地址。当你新接了一个传感器但不记得地址时，跑这段代码就能查出来：

```c
#include "main.h"
#include <stdio.h>

extern I2C_HandleTypeDef hi2c1;  // CubeMX 生成的 I2C 句柄
extern UART_HandleTypeDef huart1; // 用于打印结果

void I2C_ScanBus(void)
{
    char buf[50];
    printf("Scanning I2C bus...\r\n");

    for (uint8_t addr = 1; addr < 128; addr++)
    {
        // 尝试向地址 addr 发送一个空数据
        // HAL_I2C_IsDeviceReady 会发送起始条件+地址，检测是否有 ACK
        if (HAL_I2C_IsDeviceReady(&hi2c1, (uint16_t)(addr << 1), 2, 10) == HAL_OK)
        {
            sprintf(buf, "Device found at 0x%02X\r\n", addr);
            HAL_UART_Transmit(&huart1, (uint8_t*)buf, strlen(buf), 100);
        }
    }

    printf("Scan complete.\r\n");
}
```

::: tip I2C 地址 = 7 位地址 << 1
HAL 库的函数参数中的地址是 **左移 1 位后的 8 位地址**（最高 7 位是地址，最低位是读写位）。所以实际调用时要把 7 位地址左移 1 位：`addr << 1`。数据手册上写的通常是 7 位地址。例如 SSD1306 OLED 的 7 位地址是 0x3C，HAL 调用时传 `0x3C << 1 = 0x78`。
:::

## I2C 常见问题

**Q：通信失败，返回 HAL_ERROR 或 HAL_TIMEOUT？**

检查清单：
1. SDA 和 SCL 接对了吗？（两根线没有接反）
2. 上拉电阻有吗？（4.7kΩ 左右）
3. 设备地址对吗？（先用上面的扫描代码确认地址）
4. 两个设备共用电源了吗？（地线连在一起了吗？）
5. 线太长了吗？（I2C 不适合长距离，杜邦线超过 30cm 就不可靠了）

## 小结

I2C 用两根线（SCL + SDA）加上拉电阻，通过地址机制实现一对多通信。它是连接传感器、显示屏、存储器的最常用协议。速度不如 SPI 快，但胜在引脚少、接线简单。学会 I2C，你就能搞定大部分传感器了。
