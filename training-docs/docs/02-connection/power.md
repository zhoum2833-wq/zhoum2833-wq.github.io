# USB 到底供了多少电

## USB 口能提供什么？

你插上 USB 线给开发板供电时，那个小小的 USB 口到底能给出多少电？这个问题直接决定了你能驱动哪些东西、不能驱动哪些东西。

**USB 2.0 标准规范**明确规定：USB 口输出 **5V 直流电**，最大电流 **500mA（毫安）**。功率 = 5V x 0.5A = **2.5W（瓦）**。

这是理论值。实际使用中，大部分电脑的 USB 口可以稍微超额输出（有些主板能到 1A），但你不能依赖这个。稳妥的做法是**始终按 500mA 上限来做规划**。

::: tip 怎么知道用了多少电流？
功率 P = 电压 U x 电流 I。比如你接了一个 3.3V 的 LED（串联 220 欧姆限流电阻），电流大约 (3.3V - 2V) / 220 欧姆 ≈ 6mA。一个 LED 耗电 6mA，理论上 USB 能同时点亮 80 多个 LED。但单片机和传感器也要用电，实际留给外围的设备可能只有 200~300mA。
:::

## 5V 怎么变成 3.3V？—— LDO 降压

单片机的核心芯片（STM32、MSPM0 等）工作电压是 **3.3V**，不是 5V。所以在开发板上，USB 的 5V 需要先"降压"到 3.3V。

这个降压工作由一个叫 **LDO**（Low Dropout Regulator，低压差线性稳压器）的小芯片完成。最常见的型号是 **AMS1117-3.3**。

```
USB 5V 输入 → AMS1117-3.3 → 3.3V 稳定输出 → 单片机 VCC 引脚
                   ↑
                  GND（地线）
```

LDO 的工作原理类似一个"智能水龙头"：进水压力 5V，出水压力要保持 3.3V。如果用水量突然变大（单片机全速运行），水龙头就自动开大一点；用水量变小（单片机休眠），水龙头就关小一点。

LDO 的输入端和输出端通常各并联一个**电容**（输入端约 10uF，输出端约 22uF）。电容在这里的作用是"蓄水池"—— 过滤掉电压的波动（纹波），让输出更加平稳。

::: warning LDO 会发热
LDO 把 5V 降到 3.3V，中间的 1.7V 压差会以**热量**的形式消耗掉。电流越大，发热越严重。但单片机本身功耗很低（几十 mA），所以 LDO 只是温热，不会烫手。如果你用手摸 LDO 发现很烫，说明后端用电太多了，需要检查有没有短路。
:::

## 为什么是 3.3V 而不是 5V？

这是很多新手的疑问：USB 既然给 5V，为什么芯片不直接设计成 5V 供电？

答案在**芯片制造工艺**上。芯片内部有数以亿计的**晶体管**（电子开关），随着工艺进步，晶体管越做越小（从微米级到纳米级）。晶体管变小了，能承受的电压就必须降低 —— 就像一根头发丝粗的电线和一根手指粗的电线，能安全通过的电流不一样。

- 老的 51 单片机：5V（工艺较老，晶体管大）
- STM32（Cortex-M3/M4）：3.3V
- 手机 CPU：1V 左右（工艺极其先进，晶体管极小）

低电压的好处：**省电**（功耗和电压的平方成正比）、**发热少**、**更适合电池供电**。

## 一个 USB 口能驱动什么？

**可以驱动（总功耗 < 2.5W）**：
- 单片机本身（约 20-50mA）
- 几个 LED（每个约 5-15mA）
- 常见传感器（温度、湿度、光线、超声波等，通常几 mA 到几十 mA）
- OLED 显示屏（约 20-50mA）
- 蓝牙 / WiFi 模块（通信瞬间可能到 100-200mA，但平均不高）

**不能直接用 USB 驱动（需要独立电源）**：
- 直流电机（小电机空载也要 100-300mA，带载可能上 A）
- 舵机（堵转时可达 1-2A）
- 大功率 LED（单个 1W LED 就要 300mA）
- 继电器线圈（吸合瞬间电流很大）
- 电磁铁

::: danger 最常见的翻车现场
"我直接把两个电机的电源线焊到板子的 5V 和 GND 上，USB 供电，一开机电机转了一下，单片机就重启了，然后电脑提示 USB 端口电涌。"

这是经典的**电压跌落（Brownout）**事故。电机启动瞬间需要大电流，USB 口供不上，5V 电压被拉低到 3V 以下，单片机的供电不足 → 单片机复位重启。更严重的情况下，USB 口的过流保护可能会**永久关闭该端口**（需要重启电脑才能恢复）。
:::

## 正确的供电方案

**电机、舵机等"大胃王"设备，必须用独立电源供电。**

最简单的方案：
- USB 口 → 只给单片机 + 传感器 + 小外设供电
- 电池组（如 2S 锂电池 7.4V 或 4 节 AA 电池 6V）→ 给电机驱动板和电机供电
- **两套电源的地线（GND）必须连接在一起**（共地），否则信号无法正常传输

```
USB 5V ──────→ 开发板 3.3V ──→ 单片机
                                  │
                               信号线（PWM/GPIO）
                                  │
电池 7.4V ──→ 电机驱动板 ────→ 电机
                                  │
                      GND ────────┘  ← 地线相连（共地）
```

## 实际功耗估算练习

假设你的小车上装了这些东西，估算一下总功耗：

| 设备 | 工作电压 | 电流（大约） | 功率 |
|------|---------|-------------|------|
| STM32F103 单片机 | 3.3V | 50mA | 0.165W |
| HC-05 蓝牙模块 | 3.3V | 40mA | 0.132W |
| OLED 显示屏 | 3.3V | 30mA | 0.099W |
| 2 个 LED 指示灯 | 3.3V | 15mA | 0.05W |
| **合计** | | **135mA** | **0.446W** |

135mA，离 USB 的 500mA 上限还很远，安全。现在加上两个直流电机（每个 300mA），总共 735mA —— **超出上限了**，必须独立供电。

## 小结

USB 口提供 5V / 500mA（2.5W），经过 LDO 降为 3.3V 给单片机用。单片机 + 传感器 + 小外设安全，驱动电机和舵机必须独立供电。做功耗估算，是一个嵌入式工程师的基本习惯。
