# 为什么虚拟出 COM 口

## 历史的遗产：COM 口是什么？

在 USB 出现之前的上世纪八九十年代，电脑背后都有一个 **DB9 串口**（也叫 RS-232 口），它是一个 9 针的梯形接口，用来连接鼠标、调制解调器（Modem，拨打网络电话的"猫"）等外设。操作系统给每个物理串口分配一个名字：**COM1**、**COM2**……

到了 2026 年，你的轻薄本上早已没有这个物理接口了。但大量嵌入式设备（单片机、PLC、工业传感器）仍然用 UART 协议通信。为了让现代的 USB 电脑能和这些老协议的设备对话，操作系统想了一个办法：**用软件虚拟出一个"假"的 COM 口**。

## 虚拟 COM 口的诞生过程

你插上单片机开发板时，整个过程是这样的：

```
USB 线插入电脑
    ↓
Windows 检测到 USB 设备插入
    ↓
读取 USB转串口芯片的 VID/PID（厂家ID/产品ID）
    ↓
匹配已安装的驱动程序（如 CH340 驱动）
    ↓
驱动创建虚拟串口 → 在系统中注册为 COM3（或其他编号）
    ↓
设备管理器显示：USB-SERIAL CH340 (COM3)
    ↓
任何软件都可以像操作物理串口一样操作 COM3
```

对操作系统来说，`COM3` 就像一个真实的物理串口。任何串口软件打开 `COM3`，驱动层会把数据通过 USB 转发给 CH340 芯片，再由 CH340 转成 UART 信号发给单片机。反过来也是同样的路径。

::: tip 生活中的类比
想象你有一台老式传真机（只能接电话线），但你家里已经没有电话线接口了，只有光纤宽带。电信公司给你装了一个"光猫"，它一头接光纤，一头提供一个电话线插口。你把传真机插上去，传真机以为它接的是真电话线，其实信号已经在光猫里完成了光电转换。CH340 就是这个"光猫"。
:::

## COM 口的编号是怎么来的？

Windows 给虚拟串口分配编号时，会选一个**当前未被占用的最小编号**：

- 你插上第一个串口设备 → 分配 **COM3**（COM1 和 COM2 是系统预留的，即使没有物理串口也存在）
- 再插一个串口设备 → 分配 **COM4**
- 如果曾经插过某个设备，系统会记住它的序列号，下次再插还是同一个 COM 号

如果你插拔了不同的 USB 口，同一个设备可能被分配到不同的 COM 号。这是因为 Windows 把"设备 + USB口"的组合视为不同的连接。

## 怎么找到你的 COM 口？

这是每个新手都要掌握的基本操作：

1. **右键点击"开始"菜单**（或按 `Win + X`），选择 **设备管理器**。
2. 展开 **端口 (COM 和 LPT)** 这一项。
3. 找到类似 `USB-SERIAL CH340 (COM3)` 的条目。括号里的 `COM3` 就是你的串口号。
4. 如果没有这个条目，检查 **其他设备** 里有没有黄色感叹号的未知设备 —— 这说明驱动没装好。

::: danger 串口被占用
如果你用串口软件打开了 COM3，同时又想在另一个软件里打开 COM3，第二个软件会报错"端口被占用"或"Access Denied"。一个 COM 口同一时间只能被一个程序使用。关掉第一个软件再试。
:::

## 波特率 —— 双方约定的"聊天速度"

打开 COM 口时，除了指定端口号（如 COM3），你还需要指定**波特率（Baud Rate）**。波特率就是每秒传输多少个**位（bit）**，单位是 bps（bits per second）。

可以把波特率理解为两个人聊天的语速：

- **你说得太快**（波特率太高）→ 对方听不清，数据出错。
- **你说得太慢**（波特率太低）→ 浪费时间。
- **双方语速必须一致** → 电脑和单片机的波特率必须设成一样。

常用的波特率值：

| 波特率 | 速度 | 适用场景 |
|--------|------|----------|
| 9600 | 慢 | 调试输出、旧设备兼容、GPS 模块 |
| 115200 | 标准快 | 日常开发调试、数据传输 |
| 921600 | 很快 | 大批量数据传输（需要硬件和驱动支持） |
| 1000000+ | 极限 | 特定高速场景 |

::: tip 99% 的情况用 115200
在比赛和日常开发中，115200 是最常用的波特率。它足够快，所有设备都支持，出错概率极低。除非数据手册明确要求其他值，否则就用 115200。
:::

## 串口助手软件 —— 和单片机"聊天"的窗口

串口助手（也叫串口调试工具）是一个能在电脑上用 COM 口收发数据的软件。常用的有：

- **SSCOM**：国内老牌串口工具，轻量、功能全、支持中文。
- **Vofa+**：现代化的串口工具，支持波形显示、数据导出，非常推荐。
- **VS Code Serial Monitor**：VS Code 内置的串口监视器，适合快速查看输出。
- **PuTTY**：老牌终端软件，支持串口、SSH 等多种连接方式。

你在串口助手里选择 `COM3`、波特率 `115200`、数据位 `8`、停止位 `1`、无校验，点击"打开串口"，就成功建立了一条从电脑到单片机的通信通道。你在串口助手里打的字会变成电信号，通过 USB 线传到单片机；单片机 `printf()` 打印的内容也会出现在串口助手的接收区。

## 小结

虚拟 COM 口是操作系统为了让 USB 时代的电脑能继续和 UART 设备通信而创造出的"障眼法"。你不需要理解 USB 协议的细节，只需要知道设备的 COM 号、设置好波特率，打开串口助手，就能和单片机对话了。
