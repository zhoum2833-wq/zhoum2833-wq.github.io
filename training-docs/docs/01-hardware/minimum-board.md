# 最小系统板

## 什么是"最小系统"？

**最小系统 = 让单片机"活"起来所必需的最少元器件集合。**

一颗裸的 STM32 芯片需要至少以下四个条件才能运行你的程序：

| 必需组件 | 作用 | 没有它的后果 |
|---------|------|-------------|
| 电源 | 提供 3.3V 供电 | 芯片不工作 |
| 晶振（时钟源） | 提供精准的时钟心跳 | 芯片用内部不精确的 RC 振荡器凑合，时序不准 |
| 复位电路 | 确保上电时从第一条指令开始执行 | 上电瞬间电压不稳，CPU 可能跑飞 |
| 程序下载接口 | 把编译好的程序烧录进去 | 空芯片，什么也做不了 |

这四个条件缺一不可，所以叫"最小"——再少一个都不行。

## 为什么需要最小系统板？

STM32F103C8T6 这类芯片的物理封装是 **LQFP-48**（薄型四方扁平封装），它的引脚间距只有 **0.5mm**——不到一根头发丝的直径。

这意味着你不可能直接用面包板或杜邦线连接一颗裸芯片：
- 引脚太密，手工焊接极其困难
- 没有配套电路（晶振、电源、复位），芯片根本跑不起来
- 引脚太细，插不进面包板的 2.54mm 间距插孔

最小系统板解决的就是这个问题：它把裸芯片和最小外围电路做在一块小 PCB 上，然后把所有引脚引出到 **2.54mm（100mil）标准间距的排针**——这种间距正好能插进面包板和杜邦线的母头。

## 板子上有什么？逐一认识

假设你拿起一块 STM32F103C8T6 最小系统板（市场上最常见、最便宜的蓝色小板），从上到下看：

### 1. 单片机芯片（正中央）

黑色正方形的大芯片，表面印着 `STM32F103C8T6`。这是一颗 Cortex-M3 内核、72MHz 主频、64KB Flash、20KB SRAM 的 MCU。它是整块板子的大脑。

### 2. 晶振（旁边的小银条）

一块银色的金属壳长方形小元件，通常标着 `8.000`（8MHz）。晶振的内部是一小块石英晶体，通电后会以极其稳定的频率振动——就像音乐节拍器一样，给 CPU 提供精准的时钟节拍。

::: tip 为什么需要两个晶振？
你会发现板子上通常有两个晶振：
- **高速外部晶振（HSE）**：8MHz，为 CPU 主频提供基准时钟。经过内部 PLL（锁相环）倍频，得到最终 72MHz 系统时钟。
- **低速外部晶振（LSE）**：32.768KHz，专为实时时钟（RTC）提供基准。2^15 = 32768，刚好每秒溢出一次——用来精确计时。
:::

### 3. 复位按钮（角落的小按键）

按下它，单片机的 NRST 引脚被拉到 GND，芯片立即停止当前工作，从头开始重新执行程序。相当于电脑的重启键——但快得多。

### 4. 电源 LED

板子通电后亮起的一颗红色或绿色 LED。它只做一件事：告诉你"板子有电"。如果它没亮，别怀疑程序——先检查 USB 线是不是松了。

### 5. LDO 稳压器（通常是 1117 系列）

一个三脚的小芯片，表面标着 `1117-3.3`。它的工作是把 USB 接口进来的 5V 电压**稳稳地降到 3.3V**，供给单片机。没有它，5V 直接灌进芯片，一秒就烧。

### 6. BOOT0 跳线帽

一个塑料小帽子套在两根排针上（或一个拨动开关），控制 BOOT0 引脚的电平：

| BOOT0 | BOOT1 | 启动模式 |
|-------|-------|---------|
| 0 | 任意 | **主 Flash 启动**（正常运行模式，烧录后自动跑程序） |
| 1 | 0 | **系统存储器启动**（进入 ST 出厂固化的串口烧录程序） |
| 1 | 1 | **SRAM 启动**（调试用，很少使用） |

正常使用时，BOOT0 必须为 0。需要串口下载程序时，先把 BOOT0 拨到 1，下载完再拨回 0。

### 7. USB 接口

通常是 Micro-USB 母座（手机充电口的上一代标准）。有两个功能：
- **供电**：插上 USB 线，5V 经过 LDO 转成 3.3V 给板子供电
- **通信（在特定板子上）**：如果板子带 USB 转串口芯片（CH340），USB 还可以作为串口用于下载程序和调试信息输出

### 8. 排针（Pin Headers）

板子四周两排或者四排整齐排列的金属针，间距正好 2.54mm。每一根针都对应芯片的一个引脚，上面可能印着丝印标号（PA0、PB1、PA2……）。通过杜邦线可以把这些引脚连接到：LED、按键、电机驱动模块、传感器模块、显示屏等等。

## 最小系统板 vs 核心板 vs 开发板

| 类型 | 包含内容 | 尺寸 | 价格 | 适用场景 |
|------|---------|------|------|---------|
| **最小系统板** | MCU + 晶振 + 复位 + 电源 + 编程口 | 极小（~5x5cm） | 8~20 元 | 竞赛、产品级定制 PCB |
| **核心板** | 最小系统 + 更大 Flash/RAM + Flash 存储芯片等 | 中（~6x8cm） | 20~50 元 | 功能更强的嵌入式产品原型 |
| **开发板** | 最小系统 + 板载调试器 + LED + 按键 + 传感器 + 显示屏接口等 | 大（~10x15cm） | 80~300 元 | 学习、评估、原型验证 |

以大家最熟悉的 Arduino Uno 为例——它属于**开发板**。上面除了 ATmega328P 最小系统外，还集成了 USB 转串口调试器、一颗辅助 MCU（做 USB 转串口）、一排 LED、复位按键、电源插座……一块板子买回来插上就能玩。

## 竞赛为什么用最小系统板？

在电子竞赛中，我们几乎只用最小系统板，原因很实际：

1. **尺寸限制**：竞赛小车底盘空间有限，开发板根本塞不下
2. **重量限制**：小车越轻越好加速、好转弯
3. **成本**：一块最小系统板 10 块钱，烧了不心疼
4. **灵活设计**：我们只把需要的引脚引出来，剩余的可以自定义母板（Carrier Board）

典型做法是：设计一块竞赛专用 PCB（母板），上面把最小系统板插上去，左右两边排布电机驱动、传感器接口、显示屏、电源管理电路。这样既保证了核心 MCU 部分的可靠性（买现成的最小系统板，比自己焊裸芯片靠谱得多），又能够自由扩展竞赛所需的各种外设。

## 第一件事：检查你的板子

拿到最小系统板后，在写任何代码之前，先做三件事：

1. **目视检查**：有没有引脚歪斜、焊盘脱落？PCB 上有没有明显的烧痕？
2. **通电检查**：插上 USB 线，电源 LED 是否亮起？用万用表量一下 3.3V 和 GND 之间的电压是不是约 3.3V？
3. **连接检查**：板子插到电脑 USB 口，设备管理器里是否出现了串口设备（COM 口）？

三件事都通过，你的最小系统板就是健康的，可以开始烧录程序了。
