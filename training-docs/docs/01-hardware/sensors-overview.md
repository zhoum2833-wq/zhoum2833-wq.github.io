# 常用传感器选型概览

传感器是单片机的"眼睛"和"耳朵"。电赛题的"感知"部分全靠传感器——检测黑线、避开障碍、追踪目标、维持平衡。这一章帮你横向对比电赛中最常用的传感器，选型时不再迷茫。

|--------|--------|------|------|------|---------|

## 选型决策思路

```
需要感知什么？
  │
  ├─ 地面黑线 ──→ 8路灰度传感器（便宜够用）
  │               或 TCS34725（需要分辨颜色时）
  │
  ├─ 前方障碍物距离 ──→ 超声波 HC-SR04（2cm~4m，便宜）
  │                     或红外 GP2Y0A21（10~80cm，抗干扰）
  │                     或 ToF VL53L0X（高精度短距，I2C）
  │
  ├─ 自身旋转角度/速度 ──→ 编码器（轮子转了多少圈）
  │
  ├─ 车身姿态（倾斜/角速度）──→ JY61P（10轴，串口输出，上手快）
  │
  ├─ 电池电压/电流 ──→ 电阻分压 + ADC（电压）
  │                    INA219（电压+电流一起，省事）
  │
  ├─ 物体经过 ──→ 光电开关（对射式）
  │
  └─ 图像识别 ──→ MaixCAM Pro（跑模型，速度快）
                  OpenMV（Python 编程，上手快）
```

## 输出类型：模拟 vs 数字，选哪个？

| 输出类型 | 代表传感器 | 单片机怎么读 | 优点 | 缺点 |
|---------|-----------|-------------|------|------|
| **模拟** | GP2Y0A21、电阻分压 | ADC | 连续值、信息量大 | 易受干扰、需滤波 |
| **数字 0/1** | 红外避障、光电开关 | GPIO | 最简单 | 只有"有/无" |
| **脉冲** | 编码器、超声波 | 定时器捕获 | 精准、抗干扰 | 需要定时器资源 |
| **串口** | JY61P、MaixCAM | UART | 数据丰富 | 需要协议解析 |
| **I2C** | INA219、TCS34725 | I2C | 多设备共用总线 | 地址冲突需注意 |

## 新手常见选型错误

::: danger 常见踩坑
1. **超声波在狭窄环境里误触发**——声波打到旁边的墙壁反射回来，数据乱跳。超声波需要开阔的"视野"。
2. **红外测距对深色物体测不准**——红外光被黑色表面吸收，测出来的距离偏大。白色物体又偏小。
3. **灰度传感器离地太高**——离地 > 1cm 就基本分不出黑白了。安装高度 3~8mm 最理想。
4. **买错电压版本**——很多模块有 3.3V 和 5V 两种版本。确保和你 MCU 的 IO 电压匹配。
5. **陀螺仪上电不校准**——JY61P 上电后前几秒是校准过程，这时不能移动模块。
:::

## 成本优化

电赛预算有限，传感器不是越贵越好。一个典型巡线小车传感器成本：

| 用途 | 传感器 | 单价 | 数量 | 小计 |
|------|--------|------|------|------|
| 巡线 | 8 路灰度传感器 | ¥20 | 1 | ¥20 |
| 测速 | 霍尔编码器 | ¥15 | 2 | ¥30 |
| 姿态 | JY61P 陀螺仪 | ¥40 | 1 | ¥40 |
| **合计** | | | | **¥90** |

所有传感器总价控制在 ¥100 内是合理的。

## 小结

传感器选型是电赛方案设计的起点。先明确"需要感知什么"→ 输出类型匹配 MCU 外设（ADC/UART/I2C/GPIO）→ 安装注意物理限制（高度、视角、干扰）。买传感器前先看电压，接好滤波，做好校准。灰度 + 编码器 + 陀螺仪 = 巡线小车传感器的铁三角。
