# 单片机

**单片机**顾名思义，即**单片微型计算机**，是将一整套微型计算机系统，全部集成在单一芯片上的嵌入式控制器。正因所有功能模块都整合在一枚芯片内部，无需外接复杂电路即可独立运行，所以被称为单片机。

从硬件结构来看，单片机整体由四大核心部分构成：CPU内核、片内存储器、片上外设以及时钟、复位、电源等基础支撑电路。四部分各司其职、协同工作，共同实现单片机的运算处理、数据存储、外部交互与稳定运行。

需要特别区分一个**关键概念**：单片机内部的运算核心是CPU内核，而非完整CPU。CPU内核仅负责指令执行、数据运算、中断处理等核心逻辑功能，是纯运算IP模块；它必须搭配存储器、外设、支撑电路，才能构成一套完整、可独立工作的单片计算机系统。

接下来，我们逐一拆解讲解各部分的结构与功能。
## CPU内核

CPU内核是单片机的大脑——它负责从 Flash 里取指令、解码、执行，周而复始。你用 C 写的每一行代码，最终都变成一条条机器指令，由内核逐条执行。

你不需要把 CPU 内核想得太神秘。它本质上就是一个**按固定节奏做简单算术的逻辑电路**——只是做得极快（每秒几千万到几亿次）。

### 内核系列

当前主流单片机大多使用 ARM Cortex-M 系列内核。ARM 公司不生产芯片，只设计内核 IP，授权给 ST、TI、NXP 等厂商，各家再往内核外面加上自己的存储器、外设，做成完整的单片机。

| 内核 | 性能定位 | 代表芯片 | 电赛里干什么 |
|------|---------|---------|------------|
| **Cortex-M0/M0+** | 入门级，极低功耗 | STM32F0、STM32G0 | 简单控制、传感器读取、小项目 |
| **Cortex-M3** | 中端主力 | STM32F103（经典款） | 电赛最常用的核心，够用且便宜 |
| **Cortex-M4** | 带 DSP 和浮点单元 | STM32F4、STM32G4 | 需要做数字信号处理、PID 控制 |
| **Cortex-M7** | 高性能 | STM32H7 | 复杂算法、图像处理、高速通信 |

**电赛选手重点关注 M3 和 M4。** M3（如 STM32F103C8T6）是最经典的入门选择，资料最多、资源够用。M4（如 STM32F407）多了硬件浮点运算和 DSP 指令，如果你要做 FFT、滤波、PID 控制，M4 会更快。

### 你不需要深究内核细节

作为应用开发者，你几乎不需要直接和 CPU 内核打交道。C 编译器帮你把代码翻译成指令，调试器帮你单步跟踪，HAL 库帮你操作外设。你只需要知道：

- **主频**（如 72MHz、168MHz）：决定了单片机一秒钟能做多少事。越高越快，但功耗越大。
- **架构版本**（ARMv6-M、ARMv7-M）：决定了支持哪些指令和功能。M0/M0+ 是 ARMv6-M，M3/M4/M7 是 ARMv7-M。选型时看一眼就行，平常不用管。
