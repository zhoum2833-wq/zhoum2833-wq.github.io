# 单片机的大脑 — CPU 架构

## 什么是 ARM？

**ARM 是一家英国公司，它不生产芯片，只设计 CPU 内核，然后把设计授权卖给其他芯片公司。**

这个模式有点像：ARM 是建筑设计师，画出大楼的图纸（CPU 内核设计）；ST（意法半导体）、TI（德州仪器）、NXP（恩智浦）是建筑商，拿着图纸盖楼（制造芯片），并加入自己的外设（GPIO、ADC、定时器等）。

所以你会看到两种 ARM 的标志出现在同一颗芯片上：
- **ARM Cortex-M3**：ARM 公司设计的内核
- **STM32F103**：ST 公司围绕这个内核打造的完整芯片

## 为什么大家都用 ARM？

上世纪 90 年代，单片机市场百花齐放，每家都有自己的 CPU 架构，互不兼容。你学会了 8051，换一家公司的芯片就得重新学。

ARM 的出现改变了这一切。到今天，ARM 架构占据了嵌入式 MCU 市场的 90% 以上份额。原因很简单：

| 优势 | 说明 |
|------|------|
| 低功耗 | Cortex-M 系列专为电池供电设备设计 |
| 性能好 | 32 位架构，比 8 位/16 位竞争对手更强 |
| 生态大 | 编译器、调试器、教程、论坛，应有尽有 |
| 通用性 | 学会一种，换厂商几乎零成本，代码可移植 |

## Cortex-M 系列全家福

ARM 把 Cortex-M 系列分成几个等级，数字越大性能越强（但功耗和价格也会增加）：

| 型号 | 位数 | 主频范围 | 流水线 | 典型用途 | 是否有 FPU |
|------|------|---------|--------|---------|-----------|
| Cortex-M0 | 32 | ~50MHz | 3 级 | 遥控器、简单传感器 | 无 |
| Cortex-M0+ | 32 | ~50MHz | 2 级 | 家电控制、低端 IoT | 无 |
| Cortex-M3 | 32 | ~120MHz | 3 级 | 竞赛小车、工业控制 | 无 |
| Cortex-M4 | 32 | ~200MHz | 3 级 | 电机 FOC 控制、音频处理 | 可选 |
| Cortex-M7 | 32 | ~600MHz | 6 级 | 高端 DSP、图形处理 | 有 |

::: tip 一句话总结
- M0/M0+：简单、便宜、省电，够做基础控制
- M3：性能与成本的平衡点，竞赛最佳选择
- M4：多了 DSP（数字信号处理）和可选的 FPU（硬件浮点运算）
- M7：最快最强的 M 系列，逼近低端应用处理器
:::

## 什么是"32 位"？

**32 位意味着 CPU 一次能处理 32 个二进制位（bit）的数据。**

这有什么实际意义？看这个对比：

```c
// 8 位 MCU 只能直接处理 0~255 的整数
uint8_t  small_num = 250;   // 可以
uint8_t  big_num   = 300;   // 溢出！300 超出了 0~255 范围

// 32 位 MCU 可以直接处理 0~42 亿的整数
uint32_t small_num = 250;   // 轻松
uint32_t big_num   = 300;   // 轻松
uint32_t huge_num  = 3000000000; // 还是轻松
```

实际编程中，这意味着：
- 32 位 MCU 处理传感器数据更快（一次读 32 位，而不是分 4 次读 8 位）
- 能做更复杂的运算（比如 PID 控制中的浮点乘法累加）
- 能访问更大的内存（32 位地址线 → 最大 4GB 寻址空间）

## 什么是主频（MHz）？

**主频是 CPU 的"心跳速度"。72MHz = 每秒钟有 7200 万个时钟周期。**

打个比方：你跑步时每秒钟跨出 2 步，你的"步频"就是 2Hz。如果每跑一步就能完成一件工作（比如前进一步），你的"工作效率"就是每秒 2 个单位。CPU 也一样：每个时钟周期，它完成一小步工作。

::: warning 注意
72MHz 不等于每秒执行 7200 万条指令！

在 Cortex-M3 中，一条简单指令（如两个寄存器相加）可能只需 1 个周期，但一条复杂指令（如从内存加载数据）可能需要 2~3 个周期。所以实际指令执行速度大约是 0.5~1 倍主频。
:::

## 我们的两块芯片

### MSPM0G3507 — Cortex-M0+ @ 32MHz

这是 TI（德州仪器）推出的入门级芯片。它的内核 Cortex-M0+ 是 ARM 最精简的设计：只有 2 级流水线，指令集经过裁剪，追求最低的功耗和成本。32MHz 的主频对于循迹小车、按钮读取、LED 控制这类基础任务完全足够。它的优势是模拟外设（ADC 等）做得很出色。

### STM32F103 — Cortex-M3 @ 72MHz

这是电子竞赛中最常见的芯片。Cortex-M3 比 M0 多了硬件除法指令、更多的中断优先级、更好的调试功能。72MHz 的主频让它能从容应对 PID 控制循环、多传感器并行采集、实时数据通信等任务。学习资源也最为丰富。

::: tip 用哪个？
新手建议从 STM32F103 开始——资料最多，出问题容易找到答案。MSPM0G3507 可以作为第二块芯片来学习，拓宽视野。
:::
