# 单片机的大脑 — CPU 架构

CPU 是单片机执行指令的核心。它从内存取指令、解码、执行，周而复始。

## 位宽

CPU 的位宽决定了它一次能处理多少位数据。

| 位宽 | 单次处理范围 | 常见场景 |
|------|-------------|---------|
| 8 位 | 0~255 | 简单家电、玩具 |
| 16 位 | 0~65535 | 早期工控 |
| 32 位 | 0~42 亿 | 当前主流 MCU |

**位宽不够时，写代码会遇到溢出：**

```c
uint8_t  x = 300;   // 溢出！300 装不进 8 位
uint32_t y = 300;   // 没问题
```

位宽越大，处理同样数据需要的指令数越少，运算越快。

## 主频

主频是 CPU 的时钟频率。72MHz = 每秒 7200 万个时钟周期。

每个时钟周期，CPU 完成一小步工作。一条指令通常需要 1~3 个周期。所以 72MHz 的芯片实际每秒执行大约 3000~7000 万条指令。

主频越高，芯片越烫，耗电也越多。单片机的主频通常在几 MHz 到几百 MHz 之间。

## 内核系列

当前主流单片机大多使用 ARM Cortex-M 系列内核：

| 内核 | 主频 | 特点 |
|------|------|------|
| Cortex-M0/M0+ | ~50MHz | 最低功耗、最低成本 |
| Cortex-M3 | ~120MHz | 性能与成本的平衡 |
| Cortex-M4 | ~200MHz | 带 DSP 和可选 FPU |
| Cortex-M7 | ~600MHz | 最高性能 |

**内核是设计图纸，芯片是成品。** 同一份 Cortex-M3 图纸，ST 公司造出来叫 STM32F103，TI 公司造出来叫另一个型号——内核一样，外设不同。


## 小结

选单片机本质上是在选三样东西：**位宽、主频、内核**。位宽决定了数据处理的"车道宽度"，选 32 位是电赛的默认选择，8 位会让你处处碰壁。主频决定了"干活有多快"，72 MHz 级别的 STM32F103 对绝大多数电赛题目绰绰有余。内核系列则可以理解为"发动机型号"——同一份 Cortex-M 图纸各家芯片厂都在用，学会一个，其他的触类旁通。说到底就是这句：**内核是图纸，芯片是成品，外设是各家自己加的。**
