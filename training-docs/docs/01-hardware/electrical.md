# 引脚电气数据

## 工作电压

单片机工作电压是 3.3V。引脚的高电平是 3.3V，低电平是 0V。

| | 低电平 (LOW) | 高电平 (HIGH) |
|------|-------------|--------------|
| 输出 | ~0V | ~3.3V |
| 输入 | < 0.8V | > 2.0V |

输入电压在 0.8V~2.0V 之间时，芯片可能读成 0 也可能读成 1，结果不可靠。悬空引脚容易落在这个区间。

## 电流限制

GPIO 引脚用来传信号，不是供动力。单引脚最大输出电流约 20mA。

| 可以直接接 | 必须通过驱动电路 |
|-----------|----------------|
| LED（串 220Ω 限流电阻） | 电机 |
| 按钮、信号线 | 继电器 |
| 三极管基极 | 大功率 LED |

## 3.3V 还是 5V？—— 电平匹配的坑

很多传感器模块（尤其是 Arduino 生态的）是 **5V 电平**——它们的信号引脚输出 5V 高电平。但 STM32 的 GPIO 工作在 3.3V。

**把 5V 信号直接接到 STM32 的普通引脚上，会烧掉那个引脚——甚至烧掉整个芯片。**

部分 STM32 引脚标注了 **FT**（Five-volt Tolerant，5V 容忍），这些引脚可以承受 5V 输入。在数据手册中查 "FT" 标记——这是一个必须要养成的习惯。

| 场景 | 做法 |
|------|------|
| STM32 → 5V 设备（输出） | 大部分 5V 设备能识别 3.3V 高电平，通常直接连 |
| 5V 设备 → STM32（输入） | 查引脚是否 FT；不是 FT 就加**电平转换模块**（2/4/8 通道，几毛钱） |
| I2C 总线混用 3.3V/5V | 用 **I2C 专用电平转换**（BSS138 MOS 管方案，或成品模块） |

::: danger 最常见的烧板方式
用杜邦线把 Arduino 5V 传感器的数据脚直接插到 STM32 的 PA0 上。"反正杜邦线能插进去"——然后 PA0 就永远坏了。查数据手册 + 加电平转换，花两分钟省一块板。
:::

## 小结

你写的 `HAL_GPIO_WritePin` 最终变成引脚上的 3.3V 或 0V——代码和物理世界在这里交汇。记住两件事：**3.3V 是信号电平，不是驱动电源**（电机、继电器必须通过驱动电路）；**悬空引脚不可靠**（用上下拉电阻给它一个确定的默认状态）。每次画原理图之前过一遍这些硬约束——违反了，代码写得再好也跑不起来。
