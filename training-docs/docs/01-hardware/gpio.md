# 单片机的四肢 — GPIO 与外设一览

## 什么是引脚（Pin）？

单片机的黑色芯片四周伸出一根根银色的金属"腿"，这些就是**引脚**。每个引脚可以连接到外部的电路——传感器、LED、按钮、电机驱动芯片等等。

GPIO 是 **General Purpose Input/Output（通用输入输出）** 的缩写。"通用"的意思是：它们的功能不是固定的，由你的程序来决定每个引脚是输入还是输出。

## 两种最基本的模式

### 输出模式（Output）

你让单片机输出一个电压信号控制外部设备。只有两种状态：

| 状态 | 名称 | 电压 | 效果 |
|------|------|------|------|
| 1 | HIGH | 3.3V | LED 亮、继电器吸合、使能某个芯片 |
| 0 | LOW | 0V（GND） | LED 灭、继电器断开、禁能某个芯片 |

::: tip 这是嵌入式系统最基础的事实
**HIGH = 1 = 3.3V，LOW = 0 = 0V**

记住它，因为后面所有内容都建立在这个基础上。
:::

### 输入模式（Input）

你让单片机"读"外面的电压是高还是低。

```c
// 伪代码：读取按键状态
if (读取引脚 == HIGH) {
    // 按键松开，引脚被上拉电阻拉到 3.3V
} else {
    // 按键按下，引脚被拉到 GND
}
```

## 输出模式的两种配置

### 推挽输出（Push-Pull）

推挽就像你用两只手推一扇门：可以主动向前推（输出 HIGH），也可以主动往后拉（输出 LOW）。任何时候都有较强的驱动能力。

- 用途：驱动 LED、控制大多数数字信号
- 优点：输出强，高低电平都稳
- 是 GPIO 最常用的输出模式

### 开漏输出（Open-Drain）

开漏只有一只手——只能往下拉（输出 LOW）。想要输出 HIGH？它自己做不到，需要外部一个上拉电阻帮忙。

```
VCC (3.3V)
  |
 [R]  ← 外部上拉电阻（通常 4.7KΩ）
  |
  +----→ 引脚（同时也是输出信号线）
  |
 [MOSFET]  ← 单片机内部开关
  |
 GND
```

- 用途：I2C 通信、多个芯片共享一根信号线、5V 电平转换
- 优点：允许多个设备同时连到同一根线上而不会烧坏

## 上拉与下拉电阻

当一个引脚设为输入模式，但外部什么都没有接——悬空（Floating）——引脚上的电压是随机的。电磁干扰会让它一会儿读成 HIGH，一会儿读成 LOW。这显然不行。

**上拉电阻（Pull-Up）**：在引脚和 3.3V 之间接一个电阻（或使用芯片内置的弱上拉），让引脚"默认读 HIGH"。当有外部信号把它拉低时，才会读成 LOW。

**下拉电阻（Pull-Down）**：在引脚和 GND 之间接一个电阻，让引脚"默认读 LOW"。

::: warning 常见错误
初学者常忘了配置上拉/下拉，导致按键读取"抖动"——明明只按了一下，程序却检测到几十次按下和松开。这不是按键坏了，是引脚悬空导致的。启用内部上拉即可解决。
:::

## 外设一览

GPIO 只是引脚最基本的功能。同一个物理引脚还可以复用为各种**外设**的专用接口。下面快速认识一下你将来会用到的主要外设：

| 外设 | 全称 | 用途 | 关键特征 |
|------|------|------|---------|
| **UART** | 通用异步收发器 | 两个芯片之间串行聊天 | 两根线（TX/RX），全双工，常用于蓝牙模块、串口调试 |
| **I2C** | 集成电路总线 | 挂多个传感器/芯片 | 两根线（SCL/SDA），速率较慢，一个总线最多 127 个设备 |
| **SPI** | 串行外设接口 | 高速通信 | 四根线（MOSI/MISO/SCK/CS），速度比 I2C 快得多 |
| **ADC** | 模数转换器 | 把模拟电压转成数字 | 读取光敏、温度、红外传感器等模拟信号 |
| **PWM** | 脉宽调制 | 模拟"半电压"输出 | 快速开关，靠占空比控制电机转速、LED 亮度 |
| **Timer** | 定时器 | 精确计时和计数 | 产生精确延时、测量脉冲宽度、触发周期性任务 |

::: tip 现在不用深入理解每个外设
只需要知道它们"存在"且"各司其职"即可。当你需要用到某个外设时，再深入去看对应的教程和代码示例。没有人能一次记住所有外设。
:::

## 第一个代码示例：点亮 LED

```c
// 假设 LED 连接在 PA5 引脚
// 这是用 STM32 HAL 库的写法

// 1. 使能 GPIOA 端口的时钟
__HAL_RCC_GPIOA_CLK_ENABLE();

// 2. 配置 PA5 为推挽输出
GPIO_InitTypeDef GPIO_InitStruct = {0};
GPIO_InitStruct.Pin   = GPIO_PIN_5;       // 选择第 5 号引脚
GPIO_InitStruct.Mode  = GPIO_MODE_OUTPUT_PP; // 推挽输出
GPIO_InitStruct.Pull  = GPIO_NOPULL;       // 不上拉也不下拉
GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_LOW; // 低速即可
HAL_GPIO_Init(GPIOA, &GPIO_InitStruct);

// 3. 点亮 LED
HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET);   // HIGH → 亮

// 4. 延时 500ms
HAL_Delay(500);

// 5. 熄灭 LED
HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_RESET); // LOW → 灭
```

把这几行代码放进一个死循环里，LED 就会以 1Hz 的频率闪烁——这是全世界嵌入式工程师的"Hello World"。
