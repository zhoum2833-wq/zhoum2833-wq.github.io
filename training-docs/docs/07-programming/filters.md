# 滤波器入门

## 传感器不是完美的

你读到的传感器数据从来不会是一条平滑的线——它总是在抖动。一个距离传感器对着墙壁测距，输出可能是：

```
100, 103, 98, 105, 99, 102, 200, 101, 97, 104
                             ↑
                          野值（毛刺）
```

直接用这个值做控制 → 电机会疯掉。滤波就是在喂给 PID 之前把数据"洗干净"。

## 三种最常用的滤波方法

### 1. 移动平均滤波（Moving Average）

取最近 N 个值的平均。最简单、最暴力。

```c
#define N 8
float buf[N];
uint8_t idx = 0;

float moving_average(float new_val) {
    buf[idx % N] = new_val;
    idx++;

    float sum = 0;
    for (int i = 0; i < N; i++) sum += buf[i];
    return sum / N;
}
```

| 优点 | 缺点 |
|------|------|
| 实现简单 | 延迟大（N 越大越平滑但越慢） |
| 对周期性噪声效果好 | 对突变的野值（200 那个点）抑制不够 |
| 不挑数据类型 | 占用 N 个变量的内存 |

### 2. 一阶低通滤波（互补滤波）

最常用的软件滤波——一行公式，不占内存。

```c
// α 是滤波系数，范围 0~1
// α 越接近 1 → 跟踪越快（信新值）
// α 越接近 0 → 越平滑（信旧值）
float low_pass_filter(float new_val, float old_filtered, float alpha) {
    return alpha * new_val + (1 - alpha) * old_filtered;
}

// 使用
filtered = low_pass_filter(raw_value, filtered, 0.1f);  // α=0.1 很平滑
```

```
α=0.05：  ▁▂▃▄▅▆▇  （非常平滑，但相位滞后严重）
α=0.3：   ▂▄▅▆▇██  （平滑与响应速度的平衡）
α=0.8：   ▄▆▇█▇▆▄  （几乎跟原始信号一样，滤波效果弱）
```

::: tip 调 α 的经验
巡线灰度传感器：α = 0.1~0.2（慢速变化，要稳）
陀螺仪角速度：α = 0.05~0.1（对噪声敏感，要特别稳）
编码器速度：α = 0.3~0.5（变化快，要跟得上）
:::

### 3. 中值滤波（Median Filter）

排序后取中间值，专治野值（突然冒出来的离谱数据）。

```c
#define N 5
float buf[N];

float median_filter(float new_val) {
    static uint8_t idx = 0;
    buf[idx % N] = new_val;
    idx++;

    // 冒泡排序（N 小所以不慢）
    float sorted[N];
    memcpy(sorted, buf, sizeof(buf));
    for (int i = 0; i < N-1; i++)
        for (int j = i+1; j < N; j++)
            if (sorted[i] > sorted[j]) {
                float tmp = sorted[i];
                sorted[i] = sorted[j];
                sorted[j] = tmp;
            }

    return sorted[N/2];  // 返回中间值
}
```

中值滤波对**随机毛刺**效果极好——比如超声波传感器偶尔返回 0 或 9999，中值滤波直接把它扔掉。

## 三种滤波对比

| 滤波方法 | 平滑度 | 响应速度 | 内存占用 | 擅长 | 不擅长 |
|---------|--------|---------|---------|------|--------|
| 移动平均 | 中 | 慢 | 大（N 个变量） | 周期性噪声 | 突变信号、野值 |
| 一阶低通 | 可调 | 可调 | 小（1 个变量） | 通用 | 强烈野值 |
| 中值滤波 | 中（保持边缘） | 快 | 中（N 个变量） | 野值、脉冲噪声 | 高频噪声 |

## 实际组合使用

没有人只用一种滤波器。典型做法是**级联**：

```c
// 先中值剔除野值 → 再低通平滑
float raw = sensor_read();
float de_spiked = median_filter(raw);   // 把 200 的毛刺扔掉
float smooth = low_pass(de_spiked, prev, 0.1f);  // 再平滑
prev = smooth;
// 现在 smooth 可以喂给 PID 了
```

## 卡尔曼滤波——知道名字就够了

你会在很多地方看到"卡尔曼滤波"这个词。它确实是最优估计算法，但**入门阶段不建议自己实现**。

::: tip 卡尔曼的正确态度
卡尔曼滤波需要知道系统的数学模型和噪声协方差——这两样东西入门阶段你都没有。先用一阶低通搞定 90% 的场景，等你能感觉到"低通滤波不够用了"的时候，再回来学卡尔曼。那时候你看数学推导会有感觉得多。
:::

如果确实需要（比如平衡车姿态融合），用现成的库——MPU6050 的 DMP（Digital Motion Processor）内部就集成了姿态滤波，不需要你手写。

## 小结

三种滤波器像三种工具——移动平均是锤子（简单粗暴），一阶低通是瑞士军刀（什么都能干），中值滤波是镊子（精准拔刺）。入门阶段 90% 的滤波需求用**一阶低通（α=0.1~0.2）+ 中值滤波**的组合就能解决。卡尔曼先不碰——你暂时用不上。
