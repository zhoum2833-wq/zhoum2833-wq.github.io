# Linux

Linux 和 MCU 区别很大
1. **硬件芯片本质不同**
MCU（单片机，Cortex-M）
无 MMU 内存管理单元，没有虚拟内存；直接操作物理地址，芯片小、RAM/Flash 极小（几 KB~ 几百 KB），成本低。
运行 Linux 的 MPU（Cortex-A）
自带 MMU，支持虚拟内存、进程地址隔离；必须搭配大容量内存（64MB 起）、大容量闪存，芯片成本更高。
2. **系统与实时性**
MCU：裸机 / FreeRTOS 等轻量 RTOS，硬实时，中断、调度延迟微秒级，时序稳定可控，适合电机、传感器实时控制。
Linux：通用分时系统，默认软实时，延迟波动大，无法用于高精度运动控制。
3. **软件架构**
MCU：单地址空间，无进程隔离，一个任务崩溃整机死机；直接操作寄存器、BSP 驱动。
Linux：内核态 + 用户态隔离，多进程独立；单个应用崩溃不影响系统，有标准驱动框架、完整 Shell、海量开源软件（Python、QT、网络工具）。
4. **资源、启动、功耗**
MCU：镜像极小，上电几毫秒启动，功耗低，适合电池供电设备。
Linux：系统镜像几十上百 MB，开机几秒，功耗更高。
5. **适用场景**
MCU：传感器采集、电机伺服、便携低功耗硬件、简单控制设备。
Linux：网关、机器视觉、大屏 UI、云端联网、AI 边缘运算、多媒体设备。

## GNU/Linux 是什么

很多初学者好奇"Linux"到底指什么。严格来说，我们日常说的"Linux 系统"其实是 **GNU/Linux**：

- **Linux**：只是一个**内核**（Kernel）。内核管理硬件（CPU、内存、硬盘、网卡），提供最基本的系统服务。
- **GNU**：一套**自由软件工具集**，包括 GCC（编译器）、Bash（命令行）、Coreutils（基本命令如 `ls`、`cp`）。这些工具是操作系统"用户能用"的部分。

内核 + 工具集 = 一个完整的操作系统。就像汽车引擎（Linux）+ 方向盘、座椅、仪表盘（GNU）= 一辆能开的车。

## Linux 发行版是什么

GNU/Linux 只是"原材料"。不同的人或组织把这些原材料打包成**发行版（Distribution）**，预装不同的软件、使用不同的包管理器（安装/卸载软件的工具）。

常见的发行版：

| 发行版 | 特点 | 适合谁 |
|--------|------|--------|
| Ubuntu | 用户友好、资料最多 | 新手 |
| Arch Linux | 滚动更新、高度自定义、KISS 原则 | 喜欢折腾的用户 |
| Debian | 极其稳定、保守 | 服务器 |
| Fedora | 新技术先行 | 开发者 |

::: tip
**Arch Linux 的 KISS 原则**：Keep It Simple, Stupid（保持简单，傻瓜）。这里的"简单"不是对用户简单，而是对系统设计者简单——系统不替你做决定，由你自己掌控一切。Arch 的用户手册（Arch Wiki）被公认为最好的 Linux 文档之一。
